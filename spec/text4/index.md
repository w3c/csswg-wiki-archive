</ul><h2 id="text-spacing">text-spacing</h2>
<pre class="code">[ class-before class-after [ none | default [ [ [ min max ] | &#039;/&#039; alternate ] priority? ]? ] ]+</pre>
<pre class="code">@text-spacing-rule rule-name [ font-name [ &#039;,&#039; font-name ]* ] {
  line-head opening 0%;
  closing line-end 50%/0%;
  ideograph alphabet 25% 12.5% 50% 2;
}</pre>
<ol>
<li class="level1">The relationshipo of expansion opportunities defined here and other properties (text-justify, word-spacing, letter-spacing) is an issue</li>
<li class="level1 node">JIS/JLREQ/InDesign style (trim and specify how much to add), or specify how much to add/remove?<ol>
<li class="level2">Results differ if punctuation are proportional</li>
</ol>
</li>
<li class="level1">Spaces are not consistent for: “]&lt;big&gt;[]&lt;/big&gt;[”</li>
<li class="level1">Can &#039;none&#039; have optional min/max/alternate/priority? Does it allow expansion?</li>
<li class="level1">How to handle proportional punctuation is still an issue</li>
<li class="level1">&#039;/&#039; syntax is used only for closing+line-end, where CSS3 Text defines by the “if it fits” clause</li>
</ol>
