<h1 id="meta-update-status">Meta-Update Status</h1>
<p>
This page is here to track all the changes and meta-information we&#039;ve decided to add to our specs, and which weren&#039;t part of the original CSS3 templates. Next time you update your <abbr title="specification">spec</abbr>, check that it has all these pieces!
</p>
<ul>
<li class="level1">Title as “<abbr title="Cascading Style Sheets">CSS</abbr> Foo Module [Level N]”</li>
<li class="level1">Includes link to issues list in header (see <a href="http://dev.w3.org/csswg/css-module/" title="http://dev.w3.org/csswg/css-module/" rel="noopener">template</a>)</li>
<li class="level1">Includes link to www-style in header (see <a href="http://dev.w3.org/csswg/css-module/" title="http://dev.w3.org/csswg/css-module/" rel="noopener">template</a>)</li>
<li class="level1">Includes link to editor&#039;s draft in header (see <a href="http://dev.w3.org/csswg/css-module/" title="http://dev.w3.org/csswg/css-module/" rel="noopener">template</a>)</li>
<li class="level1">Includes <a href="http://dev.w3.org/csswg/css-module/#placement" title="http://dev.w3.org/csswg/css-module/#placement" rel="noopener">module interactions</a> and <a href="http://dev.w3.org/csswg/css-module/#values" title="http://dev.w3.org/csswg/css-module/#values" rel="noopener">values</a> sections</li>
<li class="level1">Propdef tables “Values” field name links to values section</li>
<li class="level1">Propdef tables include “Animation type” field (not “Animatable”)</li>
<li class="level1">Propdef tables include “Canonical order” field</li>
<li class="level1">Includes <a href="http://dev.w3.org/csswg/css-module/#conventions" title="http://dev.w3.org/csswg/css-module/#conventions" rel="noopener">Document Conventions</a> section</li>
<li class="level1">Includes an appropriate Conformance section</li>
</ul><h2 id="additional-conventions">Additional Conventions</h2>
<p>
There are additional editorial conventions that the CSSWG has adopted over time that you may need to apply to older drafts.
</p>
<ul>
<li class="level1">Remove explicit text “This is a normative section” (all sections are normative by default, only need to be explicit about non-normative sections).</li>
