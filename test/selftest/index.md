<h1 id="self-describing-tests">Self-Describing Tests</h1>
<p>
A self-describing test is a test page that describes what the page should look like when the test has passed. A human examining the test page can then determine from the description whether the test has passed or failed.
</p>
<div class="plugin_note plugin_note noteclassic">The following are some examples of self-describing tests, using common techniques to identify passes:
<ul>
<li class="level1"><a href="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/escapes-000.htm" title="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/escapes-000.htm" rel="noopener">Identical Renderings</a></li>
<li class="level2"><a href="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/escapes-002.htm" title="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/escapes-002.htm" rel="noopener">Green Background</a></li>
<li class="level2"><a href="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/abspos-containing-block-003.htm" title="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/abspos-containing-block-003.htm" rel="noopener">No Red 1</a></li>
<li class="level2"><a href="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/border-conflict-w-079.htm" title="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/border-conflict-w-079.htm" rel="noopener">No Red 2</a></li>
<li class="level2"><a href="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/margin-collapse-clear-007.htm" title="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/margin-collapse-clear-007.htm" rel="noopener">Described Alignment</a></li>
<li class="level2"><a href="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/table-anonymous-objects-021.htm" title="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/table-anonymous-objects-021.htm" rel="noopener">Overlapping</a></li>
<li class="level2"><a href="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/border-style-inset-001.htm" title="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/border-style-inset-001.htm" rel="noopener">Imprecise Description 1</a></li>
<li class="level2"><a href="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/text-decoration-001.htm" title="http://www.w3.org/Style/CSS/Test/CSS2.1/20100127/html4/text-decoration-001.htm" rel="noopener">Imprecise Description 2</a></li>
</ul><p>
Self-describing tests have some advantages:
</p>
<ul>
<li class="level1">They can be run easily on any layout engine.</li>
<li class="level1">They can test areas of the <abbr title="specification">spec</abbr> that are not precise enough to be comparable to a reference rendering. (For example, underlining cannot be compared to a reference because the position and thickness of the underline is UA-dependent.)</li>
<li class="level1">Failures can (should) be easily determined by a human viewing the test without needing special tools.</li>
</ul>

<p>
They also have some disadvantages:
</p>
<ul>
<li class="level1">They cannot be automated: a human must determine whether the test has passed or failed.</li>
<li class="level1">In some cases it is difficult or impossible to design the test for a glaringly obvious pass or fail. (In these cases, if it&#039;s possible to create a reference, the <a href="../../test/reftest/" title="test:reftest">reftest</a> format may be more appropriate.)</li>
</ul>

<p>
Self-describing tests must follow the <a href="../../test/format/" title="test:format">CSS test format guidelines</a>.
</p>

<p>
Additional information on writing self-describing tests is available <a href="http://www.w3.org/Style/CSS/Test/guidelines.html" title="http://www.w3.org/Style/CSS/Test/guidelines.html" rel="noopener">on the W3C site</a>.
</p>
